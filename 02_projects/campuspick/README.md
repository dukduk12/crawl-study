## 🔄 What is Dynamic Rendering?
> 웹 페이지의 데이터를 수집하거나 렌더링할 때, 서버에서 완성된 HTML을 보내주는지 혹은 브라우저에서 JavaScript가 실행되어야 데이터가 나타나는지에 따라 `정적(Static)`과 `동적(Dynamic)`으로 구분됨.

### 1️⃣ 정적 렌더링 vs 동적 렌더링
| 항목| 정적 렌더링 (Static)| 동적 렌더링 (Dynamic)| 
| --------------- | ------ | ------ |
|HTML 데이터 포함|⭕ (초기 응답에 데이터 포함)|❌ (초기 HTML은 뼈대만 존재)|
|JS 실행 필요|❌ (HTML만으로 확인 가능)|⭕ (JS가 데이터를 불러와야 함)|
|requests 라이브러리|⭕ 사용 가능|❌ 사용 불가 (데이터 수집 불가)|
|Selenium 필요 여부|❌ 불필요|⭕ 필수 (브라우저 제어 필요)|
|해당 서비스 예시|일반 뉴스 기사, 블로그 등|캠퍼스픽, 인스타그램, SPA 앱|
---
### 2️⃣ 동적 렌더링이란?
> 사용자가 페이지에 접속했을 때, 서버가 즉시 완성된 HTML을 주는 대신 JavaScript 파일을 먼저 보내고, 브라우저가 이 파일을 실행하여 나중에 화면을 그리는 방식을 의미
---
#### 💡 주요 특징
- **비동기 데이터 로딩**: 페이지를 불러온 후, API 통신을 통해 데이터를 가져와 화면에 나타남
- **UX 향상**: 전체 페이지를 새로고침하지 않고도 특정 부분의 데이터만 업데이트할 수 있음
- **크롤링의 난이도**: 일반적인 HTTP 요청(requests)으로는 비어있는 HTML만 받게 되므로, 실제 브라우저처럼 동작하는 도구가 필요

#### 🛠 기술적 대응 (Scraping)
1. `Selenium`: 실제 웹 브라우저를 띄워 JavaScript를 실행시킨 후 데이터를 수집
2. `Playwright` / `Puppeteer`: 헤드리스 브라우저를 제어하여 빠른 속도로 동적 콘텐츠를 렌더링
---
### 3️⃣ Campuspick
본 프로젝트는 `Selenium`을 활용하여 캠퍼스픽의 동적 데이터를 수집하며, 각 카테고리별 데이터 추출 진행함.
추가적으로 성능 측정을 위한 간단한 지표를 추가적으로 저장함. (`crawl_times.csv`)

---

## 🚀 Usage
1. 사전 준비 : `data/contest_categories.csv`파일 내 수집할 카테고리 ID와 이름이 준비
2. 메인 스크립터 실행
  ```bash
  python main.py
  ```
3. 결과물
  - `category_[ID].csv`: 카테고리별 상세 데이터
  - `crawl_times.csv`: 수집 단계별 성능 측정 데이터
